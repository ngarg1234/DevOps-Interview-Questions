### 3. **Service**
```yaml
apiVersion: v1
kind: Service
metadata:
  name: myapp-service
spec:
  selector:
    app: myapp
  ports:
  - port: 80
    targetPort: 80
  type: ClusterIP
```

---

### 4. **Ingress**
```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: myingress
spec:
  rules:
  - host: myapp.local
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: myapp-service
            port:
              number: 80
```

## ðŸ§  Smart Tips to Remember

- **Always start with `apiVersion`, `kind`, and `metadata`.**
- **Use `kubectl explain <resource>`** to see structure.  
  Example:
  ```bash
  kubectl explain pod.spec.containers
  ```
- Use [K8s manifest generators](https://k8syaml.com/) or `kubectl create` with `--dry-run=client -o yaml`
